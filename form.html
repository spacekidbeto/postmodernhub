Goal

If any form field is empty, your program should make the div.error box appear. Update the div.error p.errors node with an appropriate error message. (eg. "First name is blank. Last name is blank." etc.)

If all the fields are complete, your program should replace the blanks in the big header with the full name and favorite food, as specified by the values in the input fields.

Stick with pure JavaScript and do not use a library.

Notes & Tips

You will notice that the form does not submit anywhere, and this is intentional. The event object's preventDefault() method causes this. In class, I had been using return false, which I now realize is outdated (this used to be the standard approach).

I have extracted the validateForm function out of the inline location shown in class. This would have been the next step, if I had not run us out of time.

You merely need to complete the implementation of validateForm. Note, however, that this really should be split into multiple functions: one that validates the form, and a second that updates the display. Bonus points if you use multiple functions.

HTML documents let you getElementById, span elements let you manipulate their innerHTML, and input elements have a value property.

Do not change the HTML body. It is intended to coerce you into exploring specific JS methods.
